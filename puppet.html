Puppet Research Projects

During the Summer of 2016, I worked as a UX intern at Puppet. Puppet is a company that makes IT automation software for managing and securing your infrastructure. As an intern, I facilitated over 30 usability test sessions and used a variety of different research methods. I conducted a heuristic evaluation, facilitated A/B testing of global search concepts, and evaluated various features through moderated usability testing. Due to my non-disclosure agreement, I can only share the results of features already released. Below is a case study on the usability of the Puppet Enterprise installer.

The Challenge

Improve the usability of the Puppet Enterprise installer by conducting usability test sessions with internal and external participants. I worked with a UX Designer on this project.

Goals

  • Assess how well users are able to install in text-mode and recover from failed installations to meet product initiatives.
  • Find usability issues and areas for improvement to influence the future direction of the installer.

Usability Issues

We selected three internal users for the first round of usability testing on the newest version of the Puppet Enterprise installer. To successfully install Puppet Enterprise, the user needs to set a console admin in the installer configuration file. Our findings showed that users were unaware they needed to set a password and the error message thrown was ineffective for troubleshooting the issue.

All participants encountered the error message below which is displayed when there is a missing console admin password in the configuration file. Users interpreted the message as an issue with curly braces on line 33.

When participants reviewed the configuration file, they would ignore the “console_admin_password” field to focus on the formatting of line 33. The error message was ineffective at pointing the users towards the correct solution. Participants were unable to troubleshoot and complete installation without being prompted.

Solutions

My findings showed that entering the admin password was an issue for all participants so changes should be made to make the process more obvious. This could be achieved either by using color to make the line more visible or a larger comment field. I also made the recommendation to update the error message so the user is notified of a missing console admin password. These findings resulted in the improved error message below.

We were able to test and validate this solution during the subsequent external usability test sessions with five active Puppet users from various organizations. Users were selected based on the type of installer they were using. We chose two Puppet open source users who have never used the official installer, and three who were actively using Puppet Enterprise. The improved error messaging caused participants to recover quickly and without prompting.

However, the error still needed to be prevented in the first place. Based off my recommendations, a new design was implemented for increased visibility of the console admin password field that was validated during the installer portion of the benchmarking sessions. Overall, the goals for the study were met and users were successful at installing Puppet Enterprise.
